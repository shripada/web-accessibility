Color Contrast Issues Demo

Purpose: This page demonstrates common color contrast accessibility violations.

These examples intentionally violate WCAG AA guidelines (4.5:1 for normal text, 3:1 for large text and UI components).

Use this page to: Test with accessibility tools, screen readers, or try viewing with different visual conditions.

Normal Text - AA Violations (Need 4.5:1)

Light Gray on White (Contrast: ~2.8:1)

This text uses #999999 on #ffffff background. This is very hard to read for people with low vision or color blindness. The contrast ratio fails WCAG AA standards significantly.

Yellow on White (Contrast: ~1.07:1)

This bright yellow text (#ffff00) on white is nearly impossible to read. This is one of the worst possible color combinations for accessibility.

Very Light Gray on White (Contrast: ~1.6:1)

This text uses #d3d3d3 on white. Many users, especially those with visual impairments, will struggle to read this content.

Sky Blue on White (Contrast: ~2.1:1)

This light blue text (#87ceeb) fails to meet minimum contrast requirements. It's especially problematic for users with color vision deficiencies.

Large Text - AA Violations (Need 3:1)

Large Bold Text with Poor Contrast (#bbbbbb on #ffffff - Contrast: ~1.9:1)
This large text still fails AA standards despite being bigger and bold.
Orange on Light Yellow Background (Contrast: ~1.4:1)
Even though this is large text, the contrast is far too low.
Pink Text on White Background (Contrast: ~1.95:1)
Pink (#ffc0cb) lacks sufficient contrast even for large text.

Buttons with Poor Border Contrast

Light gray background with buttons that have borders barely visible:

Border #e0e0e0 on #f0f0f0 background - nearly invisible
Border #dddddd on #f0f0f0 + gray text - multiple violations
Border #ffffe0 on #fffacd background with #ffd700 text - everything blends together
Note: UI components like buttons, form inputs, and focus indicators need at least 3:1 contrast ratio against adjacent colors per WCAG 2.1 AA (Success Criterion 1.4.11).

Dark Background Violations

Dark Gray Text on Darker Gray Background (Contrast: ~1.5:1) - This text is very difficult to read.
Slightly Lighter but Still Poor Contrast (Contrast: ~1.8:1) - Still fails accessibility standards.
Border and text both fail contrast requirements

Link Contrast Issues

This paragraph contains a link with very poor contrast that is hard to distinguish from regular text.

Another example with a link that has no underline and poor contrast making it nearly impossible to identify as clickable.

Note: Links must be distinguishable from surrounding text by more than just color. If relying on color alone, the contrast difference should be at least 3:1, plus a non-color indicator (like underline) should appear on focus/hover.

Testing Tools:

  • Use browser DevTools (Lighthouse, Accessibility panel)
  • Try the WAVE browser extension
  • Use color contrast analyzers like WebAIM's Contrast Checker
  • Test with different display settings and color filters
  • Simulate color blindness to see the real impact
