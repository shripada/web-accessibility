Visually Hidden Content for Accessibility

What is Visually Hidden Content?

Visually hidden content is text that is hidden from sighted users but remains accessible to screen readers. This technique is useful for providing additional context or labels to icon-only buttons, without cluttering the visual design.

The .visually-hidden class uses CSS to remove content from the visual display while keeping it in the DOM and accessible to assistive technologies.

Key Benefits

  • Screen Reader Accessible: Assistive technology users get clear, descriptive labels for interactive elements
  • Clean Visual Design: Sighted users see only the icons, keeping the interface minimal and modern
  • Keyboard Accessible: The hidden text ensures that keyboard navigators understand what each button does
  • Better Context: Providing full descriptive text (like "access help" instead of just "help") gives more context in rapid screen reader navigation

How the CSS Works

The .visually-hidden class uses several CSS properties:

position: absolute; - Removes the element from document flow
width: 1px; height: 1px; - Makes it virtually invisible
margin: -1px; - Ensures no space is taken up
overflow: hidden; - Hides any overflow content
white-space: nowrap; - Prevents text wrapping
clip: rect(0 0 0 0); - Legacy clipping for older browsers
clip-path: inset(50%); - Modern clipping method

Tips

  • Use visually-hidden text to provide additional context that sighted users might infer visually (like colors or proximity)
  • Don't use display: none; or visibility: hidden; for accessibility labels - these hide content from screen readers too
  • Test with actual screen readers to ensure your hidden text is properly announced
  • Use descriptive verbs in hidden text (e.g., "open settings" instead of "settings") to clarify what action the button performs
